# Mutation Guardrails

- **Propose before major rewrites** — for large or structural changes, show a change plan and get alignment before applying.
- **Prefer minimal edits** — update only what is necessary; append or patch rather than replace when possible.
- **Avoid duplication** — do not copy the same content across multiple artifacts; link or reference instead.
- **Do not hallucinate facts** — if information is missing, ask or mark as “Needs confirmation.”
- **If unsure, mark as “Needs confirmation”** — never invent alignment, metrics, or commitments.
- **Avoid duplicate decision entries on rerun** — check for existing entries; update or skip rather than appending again.
- **Never auto-advance lifecycle states** — do not move work to “Done” or change phase without explicit user intent.
- **Never silently change scope** — scope changes require a logged decision and visible update.
